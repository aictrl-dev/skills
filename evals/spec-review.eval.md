# Eval: spec-review

## Scenario

Provide an issue that requests a new authenticated API field but omits tenant
authorization, UI type synchronization, negative tests, and migration behavior.

## Pass criteria

- [ ] Inspects repository code and tests rather than reviewing prose alone.
- [ ] Maps every stated criterion to code impact and verification evidence.
- [ ] Flags all seeded authorization, type-sync, negative-test, and migration gaps.
- [ ] Classifies findings by severity and supplies concrete replacement criteria.
- [ ] Returns READY, READY WITH MINOR EDITS, or NOT READY with justification.
- [ ] Does not edit the issue or implement code without explicit permission.
